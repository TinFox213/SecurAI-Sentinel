# System Architecture Overview

This document provides a conceptual, high-level overview of the SecurAI Sentinel system architecture. It covers the frontend user interface components, backend infrastructure, and database schemas.

## Frontend UI Components and Layout

The frontend of SecurAI Sentinel is built as a single-page web application using a modern UI layout, fluid transitions, and a modular interface structure.

### User Interface Layout and Aesthetics
- Core Shell: The interface features a persistent navigation sidebar that controls the active workspace module, a global command palette for quick search, and header metrics including AI provider status indicators.
- Styling System: Custom styling tokens define a premium dark-themed environment. Color coding indicates threat levels (rose for Critical/High, amber for Medium, and cyan or emerald for Low/Safe status).
- Animation and Transitions: Interactive dashboards utilize smooth, hardware-accelerated animations to show changes in network traffic state and visual alerts.

### Modular Registry
- Application Routing: A centralized module registry maps navigation keys to components. Each module has defined attributes such as target workflows, secondary interest groups, primary team labels, and specific visual accents.
- Feature Isolation: The main controller reads the navigation state and renders the matching component, passing down saved settings and history references.

### Visualization Frameworks
- Trend Analysis: Graphic components plot chronological threat indexes and security posture score histories over time.
- Node Diagrams: Interconnected components map red-team activities or defensive topologies. For example, the MITRE ATT&CK Mapper uses nodes to display active tactics, techniques, and procedures in an interactive graph.

### Payload Security
- Client-Side Encryption: Before sending raw scan data (such as logs or message headers) to the backend, the frontend serializes and encrypts the payload using symmetric key encryption (AES-256).
- Secret Management: A pre-shared key acts as a secure transit wrapper, protecting raw threat details from local interception during transmission.

## Backend Infrastructure

The backend serves as the core orchestration and execution engine. It handles communication security, coordinates local networking audits, schedules alerts, and manages upstream AI integrations.

### API Security and Access Rules
- Auth Gate: The API validates requests using token-based authentication. If a token is configured, all incoming requests must supply it.
- Local Isolation: If no token is set, the server restricts access to the local loopback interface (localhost) to prevent unauthorized remote requests.
- Origin Validation: CORS controls are strictly bound to local origins unless explicitly configured otherwise.

### Asynchronous Event Engine
- Real-Time Messaging: A WebSocket server maintains live connections with active web clients.
- Alert Broadcast: An event emitter intercepts warnings generated by background processes (such as port alerts and honeypot triggers) and broadcasts them instantly to all connected UI clients.

### Socket Observers
- Ghost Ports: Background TCP sockets bind to unauthorized ports. When a connection attempt occurs (such as a local port scan), the socket registers the remote IP address, target port, and timestamp, pushes a warning to the alert history, and closes the connection.
- Address Resolution Protocol (ARP) Watchdog: Periodically monitors the local system network table, resolves the default network gateway MAC address, and tracks changes to detect spoofing or poisoning attempts.

### AI Integration Adapter
- Provider Routing: The API handles communication with various large language models (such as Gemini, OpenRouter, or NVIDIA AI services).
- Fallback Sequence: The system runs checks to identify available API keys and routes prompts to the corresponding service. In case of timeouts or failures, it moves to the next configured provider.
- Structured Parsing: System prompts demand responses formatted in clean text structures, which the adapter parses into threat scores, findings lists, and recommendations.

## Database Schemas

The application relies on browser-side IndexedDB databases for local storage. This maintains confidentiality, as all collected scan histories and configuration profiles remain in the user's local browser storage.

### Data Tables and Structural Entities

The storage database consists of several primary entities:

- Scan History
  - Purpose: Stores historical text scans, audit targets, and associated analysis results.
  - Fields: Unique identifier, timestamp, scan type, raw text input, threat level classification, risk rating, detailed summary text, mitigation recommendations list, and MITRE mapping list.

- Known Devices
  - Purpose: Tracks local network assets discovered during sweeps.
  - Fields: IP address, MAC address, host name, hardware manufacturer/vendor, status state (online/offline), and timestamp.

- Settings
  - Purpose: Persists workspace configurations and custom assets.
  - Fields: Option key, configuration values, saved playbooks, watchlist items, and interface settings.

- Endpoint Telemetry
  - Purpose: Simulates endpoint data collection for enterprise hosts.
  - Fields: Agent identifier, device name, operating system, average CPU/RAM load, agent connection status, isolation state (isolated/connected), and recent system events.

- Dark Web Scans
  - Purpose: Stores records of domain and email leak lookups.
  - Fields: Target string, query type (domain/email), leak counts, lists of compromised services, and verification timestamp.

- Posture History
  - Purpose: Tracks security posture scores over time.
  - Fields: Timestamp, overall posture score, category ratings (network, endpoint, data, operations), and count of outstanding recommendations.

- Forensics Events
  - Purpose: Aggregates alerts and logs into a chronological investigation trail.
  - Fields: Unique identifier, event timestamp, source module, severity level, description message, target system details, and association keys.
